# Подробная инструкция по настройке и 10-минутной проверке AI-кластера

Данная инструкция содержит подробное описание процесса сбора необходимых токенов, настройки внешних сервисов (Ngrok, Cloudflare, Yandex Disk WebDAV), связывания двух аккаунтов Kaggle в замкнутый цикл авто-ротации, а также сценарий быстрого 10-минутного тестирования для верификации бесшовной работы.

---

## 1. Сбор токенов и настройка внешних интеграций

### А. Telegram Bot (Интерфейс управления)
1. Откройте Telegram и найдите официального бота **@BotFather**.
2. Отправьте команду `/newbot` для создания нового бота.
3. Следуйте инструкциям: введите отображаемое имя бота, а затем системное имя (username), заканчивающееся на `_bot` (например, `MyKaggleClusterBot`).
4. **@BotFather** выдаст вам строку API токена (формата `123456789:ABCdefGhIJKlmNoPQRsTUVwxyZ`). Скопируйте её. Этот токен будет общим для всех нод кластера.

### Б. Ngrok (Публичные туннели)
1. Зарегистрируйтесь или авторизуйтесь на сайте [ngrok.com](https://ngrok.com/).
2. В левой панели управления перейдите в раздел **Getting Started** -> **Your Authtoken**.
3. Скопируйте строку вашего персонального токена.
4. *(Опционально)* Если вы хотите использовать постоянный адрес туннеля, перейдите в раздел **Cloud Edge** -> **Domains**, создайте бесплатный статический домен и скопируйте его имя (например, `sabbath-lucid-molar.ngrok-free.dev`).
5. **Важно для 24/7 работы**: Поскольку во время переключения (Handover) старая и новая ноды работают одновременно в течение 1–2 минут, использование одного токена Ngrok вызовет ошибку лимита одновременных сессий. Рекомендуется использовать два разных аккаунта Ngrok:
   - Токен от первого аккаунта пропишите в секреты **Account A** под именем `NGROK_AUTHTOKEN` (для LLM на порту 8080) и `NGROK_AUTHTOKEN_2` (для медиа-сервера на порту 8081).
   - Токен от второго аккаунта пропишите в секреты **Account B**.

### В. Cloudflare Worker + KV (Статический маршрутизатор)
1. Авторизуйтесь на сайте [dash.cloudflare.com](https://dash.cloudflare.com/).
2. В боковом меню перейдите в раздел **Workers & Pages**.
3. Создайте хранилище KV (Ключ-Значение) для адресов:
   - В меню подраздела нажмите на **KV**.
   - Нажмите синюю кнопку **Create a Namespace**. Назовите базу `KAGGLER_ROUTER_KV` и нажмите **Create**.
4. Создайте сам Worker:
   - Перейдите в **Workers & Pages > Overview**.
   - Нажмите **Create Application** -> **Create Worker**.
   - Назовите воркер (например, `kaggle-router`) и нажмите **Deploy**.
   - На странице воркера нажмите кнопку **Edit Code** (или **Quick Edit**), очистите стандартный шаблон и вставьте весь код из файла **[scripts/cf_worker.js](file:///d:/123VsakayaVsyachina/___LAB/AI_WEB/_RestrictAI/Kaggle/KeglaAI/kaggle-llm-server/scripts/cf_worker.js)**. Нажмите **Save and Deploy**.
5. Привяжите KV и секрет безопасности:
   - Вернитесь на панель управления воркера `kaggle-router`, выберите вкладку **Settings** (наверху).
   - В разделе **KV Namespace Bindings** нажмите **Add binding**. Введите:
     - Variable name (имя переменной в коде): `KAGGLER_ROUTER`
     - KV Namespace: выберите из выпадающего списка `KAGGLER_ROUTER_KV`.
     - Нажмите **Save**.
   - В разделе **Environment Variables** нажмите **Add variable** для создания токена авторизации:
     - Key: `HANDOVER_SECRET`
     - Value: Любая секретная строка (например, `my_super_secret_token_123`). Нажмите **Save**.
6. Скопируйте итоговый адрес воркера (например, `https://kaggle-router.my-username.workers.dev`). Любые внешние программы будут обращаться по этому адресу, а Cloudflare Worker автоматически перенаправит их на активную ноду Kaggle.

### Г. Yandex Disk (Синхронизация базы переписки и MCP)
Синхронизация SQLite базы данных `agent.db` происходит через протокол WebDAV. Это самый стабильный способ синхронизации без необходимости сложной авторизации OAuth2 (как в Google Drive), так как все параметры задаются через переменные окружения.
1. Авторизуйтесь на Яндексе.
2. Перейдите в настройки безопасности Яндекс ID: [passport.yandex.ru/profile/phones](https://passport.yandex.ru/profile/phones) -> раздел **Пароли приложений** (или "Пароли и авторизация").
3. Нажмите **Создать пароль приложения**, выберите категорию **Файлы (Диск)**. Назовите пароль, например, `KaggleSync`.
4. Яндекс выдаст вам 16-значный пароль приложения вида `abcd-efgh-ijkl-mnop`. Скопируйте его.
5. Ваши реквизиты для синхронизации: провайдер `yandex`, логин — ваша почта Яндекс, пароль — скопированный код приложения.

## 2. Настройка секретов в Kaggle (Circular Chain)

Для создания циклической цепочки 24/7 нам потребуются два аккаунта Kaggle. Назовем их **Account A** и **Account B**.

1. На **обоих** аккаунтах создайте новый Kaggle Notebook с именем `keglaai`. Это важно, чтобы адрес ноутбука оканчивался на `/keglaai` (slug).
2. Зайдите в настройки аккаунта Kaggle (клик по аватарке внизу слева -> **Settings**). Листайте до раздела **API** и нажмите **Create New Token**. Скачается файл `kaggle.json`.
   - Откройте скачанный файл с **Account A**, скопируйте `username` и `key`. Они понадобятся как целевые секреты на **Account B**.
   - Откройте скачанный файл с **Account B**, скопируйте `username` и `key`. Они понадобятся как целевые секреты на **Account A**.
3. В редакторе ноутбука на обоих аккаунтах откройте меню **Add-ons > Secrets** и пропишите следующие ключи:

### Карта секретов Kaggle

| Имя секрета (Secret Key) | Значение на **Account A** | Значение на **Account B** | Описание |
| :--- | :--- | :--- | :--- |
| `TELEGRAM_BOT_TOKEN` | *Токен вашего Telegram-бота* | *Токен вашего Telegram-бота* | Бот, созданный через BotFather |
| `HANDOVER_SECRET` | `my_super_secret_token_123` | `my_super_secret_token_123` | Должен совпадать с `HANDOVER_SECRET` в CF Worker |
| `CF_WORKER_URL` | `https://kaggle-router.my-username.workers.dev` | `https://kaggle-router.my-username.workers.dev` | URL-адрес вашего воркера |
| `RCLONE_PROVIDER` | `yandex` | `yandex` | Провайдер облачного бэкапа |
| `RCLONE_USER` | `ваша_почта@yandex.ru` | `ваша_почта@yandex.ru` | Логин к Яндекс Диску |
| `RCLONE_PASS` | `abcd-efgh-ijkl-mnop` | `abcd-efgh-ijkl-mnop` | Пароль приложения Диска |
| `NGROK_AUTHTOKEN` | *Ngrok Токен 1 от Account A* | *Ngrok Токен 1 от Account B* | Токен туннеля для GPU 0 (Порт 8080) |
| `NGROK_AUTHTOKEN_2` | *Ngrok Токен 2 от Account A* | *Ngrok Токен 2 от Account B* | Токен туннеля для GPU 1 (Порт 8081) |
| `ROTATION_TIME_SECONDS` | `600` | `600` | **Тестовый лимит**: 10 минут (600 секунд) вместо 9 часов |
| `NEXT_KAGGLE_USERNAME` | `username_от_Account_B` | `username_от_Account_A` | **Имя целевого аккаунта** (указывает на соседа) |
| `NEXT_KAGGLE_KEY` | `api_key_от_Account_B` | `api_key_от_Account_A` | **API Ключ целевого аккаунта** (из kaggle.json соседа) |
| `NEXT_KAGGLE_SLUG` | `keglaai` | `keglaai` | Название целевого ноутбука (slug) |

## 3. Сценарий проверочного 10-минутного тестирования

Чтобы убедиться, что вся цепочка работает корректно и без ошибок, выполните тест по шагам:

1. **Запуск первой ноды**: Откройте ноутбук на **Account A** и нажмите **Run All** (Запустить всё).
2. **Первоначальный старт**:
   - Ноутбук скачает репозиторий с GitHub.
   - Инициирует `rclone_sync.sh download` (так как на Яндекс Диске файлов еще нет, создастся чистая пустая база данных `data/agent.db`).
   - Соберет или распакует prebuilt-архив `llama.cpp`.
   - Запустит `llama-server` на GPU 0 и FastAPI `media_server.py` + Telegram-бота + таймер авто-ротации на GPU 1.
   - Зарегистрирует публичные адреса ноды в Cloudflare KV через POST `/register`.
3. **Проверка связи**:
   - Откройте Telegram, зайдите в чат с вашим ботом и напишите `/start`. Бот должен поприветствовать вас на русском языке.
   - Напишите команду `/status`. Бот должен прислать подробный статус работы кластера, где оба порта (8080 и 8081) находятся в зеленой зоне.
4. **Ожидание ротации**: Скрипт `failover_timer.py` запустит обратный отсчет на 10 минут (600 секунд).
5. **Автоматический запуск Account B**:
   - Ровно через 10 минут таймер на Account A проснется.
   - Скрипт подставит учетные данные Account B в файл `~/.kaggle/kaggle.json`.
   - Выполнит команду `kaggle kernels push`, отправляя команду на старт ноутбука на **Account B**.
6. **Подключение Account B**:
   - На панели Kaggle у **Account B** запустится сессия.
   - Account B вызовет `rclone_sync.sh download` и скачает из Яндекс Диска файл `agent.db` (база будет содержать историю ваших сообщений с шага 3 и белый список пользователей).
   - Запустит свои нейросети, обновит KV в Cloudflare Worker новыми ссылками.
7. **Передача эстафеты (Handover)**:
   - Новая нода (Account B) обратится к старой ноде (Account A) через REST API по адресу `/v1/handover/complete`.
   - Нода Account A получит сигнал:
     1. Запустит финальный бэкап `rclone_sync.sh upload`, выгружая последние изменения базы данных на Яндекс Диск.
     2. Мягко погасит процессы `llama-server`, `telegram_bot`, `failover_timer` и туннели.
     3. Запишет файл-метку `/tmp/handover_complete`.
   - Ячейка `Keep-alive` в ноутбуке **Account A** обнаружит этот файл, завершит свое выполнение, и сессия Kaggle закроется. Расход лимитов GPU на Account A мгновенно прекратится.
8. **Продолжение диалога**:
   - Напишите любое сообщение боту в Telegram. Бот ответит вам, используя контекст из скачанной базы.
   - Перенаправление произошло абсолютно бесшовно через Cloudflare Worker, при этом история переписки не потерялась.
9. **Замыкание цикла**: Еще через 10 минут Account B разбудит Account A, и круг повторится.
