# Инструкция по настройке и проверке AI-кластера (Cloudflare Tunnel + Yandex Disk)

Данная инструкция содержит пошаговое описание настройки стабильной 24/7 инфраструктуры ИИ-кластера на базе **Cloudflare Tunnel (Zero Trust)** с собственным доменом и **Яндекс Диска (WebDAV)** для синхронизации базы данных и передачи сигналов ротации между аккаунтами Kaggle.

---

## 1. Настройка внешних интеграций

### А. Telegram Bot (Интерфейс управления)
1. Откройте Telegram и найдите официального бота **@BotFather**.
2. Отправьте команду `/newbot` для создания нового бота.
3. Введите отображаемое имя бота, а затем юзернейм, оканчивающийся на `_bot` (например, `MyKaggleClusterBot`).
4. Скопируйте выданный API токен (формата `123456789:ABCdefGhIJKlmNoPQRsTUVwxyZ`). В секретах Kaggle он будет называться **`TELEGRAM_BOT_TOKEN`**.

### Б. Cloudflare Tunnel (Zero Trust) — Туннель со своим доменом
Этот шаг связывает ваш домен с локальными портами на Kaggle без использования сторонних прокси-серверов.
1. Зарегистрируйте бесплатный аккаунт на [cloudflare.com](https://www.cloudflare.com/) и добавьте туда свой домен (например, `youngfreak.click`), переведя NS-серверы домена на Cloudflare.
2. В левой панели Cloudflare перейдите в раздел **Zero Trust**.
3. В меню Zero Trust выберите **Networks** -> **Tunnels** и нажмите **Create a Tunnel**.
4. Выберите тип туннеля **Cloudflared**, нажмите Next. Назовите туннель (например, `kaggle-cluster`) и сохраните его.
5. На странице установки под заголовком «Choose your environment» выберите любой вариант и скопируйте длинный токен туннеля из строки команды (это строка символов после слова `--token`). В секретах Kaggle этот токен будет называться **`CLOUDFLARE_TUNNEL_TOKEN`**.
6. Перейдите во вкладку **Public Hostname** внутри настроек вашего туннеля на Cloudflare и добавьте две записи:
   * **Запись 1 (для API)**:
     * Subdomain: `api`
     * Domain: `youngfreak.click`
     * Type: `HTTP`
     * URL: `localhost:8080`
   * **Запись 2 (для картинок и видео)**:
     * Subdomain: `media`
     * Domain: `youngfreak.click`
     * Type: `HTTP`
     * URL: `localhost:8081`

### В. Yandex Disk WebDAV (Синхронизация истории и сигналов)
Синхронизация SQLite базы данных `agent.db` и передача сигнала завершения старой ноды происходит через облачное хранилище Яндекс Диска.
1. Авторизуйтесь под вашим аккаунтом на Яндексе.
2. Перейдите в Яндекс Паспорт -> [Управление аккаунтом](https://passport.yandex.ru/profile/phones) -> **Пароли приложений**.
3. Нажмите **Создать пароль приложения**, выберите тип **Файлы (Диск)**. Назовите пароль (например, `KaggleSync`).
4. Скопируйте сгенерированный 16-значный пароль приложения (формата `abcd-efgh-ijkl-mnop`). В секретах Kaggle он будет называться **`RCLONE_PASS`**.

## 2. Настройка секретов в Kaggle (Circular Chain)

Для создания циклической цепочки 24/7 потребуются два аккаунта Kaggle: **Account A** и **Account B**.

1. На **обоих** аккаунтах создайте новый Kaggle Notebook с именем `keglaai` (чтобы адрес ноутбука оканчивался на `/keglaai` — это slug).
2. Зайдите в профиле каждого аккаунта в **Settings > API** и нажмите **Create New Token**. Скачается файл `kaggle.json`.
   * Откройте файл от **Account A**, скопируйте `username` и `key`. Они понадобятся как целевые секреты на **Account B**.
   * Откройте файл от **Account B**, скопируйте `username` и `key`. Они понадобятся как целевые секреты на **Account A**.
3. В редакторе ноутбука на обоих аккаунтах откройте меню **Add-ons > Secrets** и пропишите следующие ключи:

### Таблица секретов в Kaggle

| Имя секрета (Secret Key) | Значение на **Account A** | Значение на **Account B** | Описание |
| :--- | :--- | :--- | :--- |
| `TELEGRAM_BOT_TOKEN` | *Токен вашего Telegram-бота* | *Токен вашего Telegram-бота* | Бот из @BotFather |
| `HF_TOKEN` | *Ваш токен Hugging Face* | *Ваш токен Hugging Face* | Токен для скачивания моделей |
| `CLOUDFLARE_TUNNEL_TOKEN` | *Токен туннеля Cloudflare* | *Тот же токен туннеля Cloudflare* | Общий токен туннеля Zero Trust |
| `RCLONE_PROVIDER` | `yandex` | `yandex` | Укажите значение `yandex` |
| `RCLONE_USER` | `ваша_почта@yandex.ru` | `ваша_почта@yandex.ru` | Логин к Яндекс Диску |
| `RCLONE_PASS` | `abcd-efgh-ijkl-mnop` | `abcd-efgh-ijkl-mnop` | Пароль приложения Диска |
| `HANDOVER_SECRET` | `youngfreak_secret` | `youngfreak_secret` | Любое секретное проверочное слово |

| `NEXT_KAGGLE_KEY` | `api_key_от_Account_B` | `api_key_от_Account_A` | **API Ключ целевого соседа** (из kaggle.json) |
| `NEXT_KAGGLE_SLUG` | `keglaai` | `keglaai` | Имя целевого ноутбука (`keglaai`) |

## 3. Сценарий проверочного 10-минутного тестирования

После того как вы настроили секреты, запустите проверку работы циклического кластера:

1. **Запуск первой ноды**: Откройте ноутбук на **Account A** и запустите все ячейки.
2. **Первоначальный старт**:
   - Ноутбук скачает репозиторий с GitHub.
   - Инициирует `rclone_sync.sh download` (базы данных на Диске еще нет, создастся чистая локальная БД `data/agent.db`).
   - Соберет/распакует бинарники нейросети, запустит LLM на GPU 0 и медиа-сервер на GPU 1.
   - Подключит туннель Cloudflare по токену `CLOUDFLARE_TUNNEL_TOKEN`. Домены `api.youngfreak.click` и `media.youngfreak.click` станут доступны.
3. **Проверка связи**:
   - Откройте Telegram и напишите боту `/start`, а затем `/status`.
   - Бот должен прислать подробный зеленый статус, подтверждающий работу модели на вашем домене.
4. **Запуск второй ноды (Account B)**:
   - Ровно через 10 минут таймер на Account A проснется.
   - Подставит ключи Account B в `~/.kaggle/kaggle.json` и выполнит `kaggle kernels push`.
   - Ноутбук на **Account B** запустится в облаке.
5. **Синхронизация и Передача (Handover)**:
   - Account B при старте скачает БД `agent.db` из Яндекса (сохраняя всю историю сообщений).
   - Запустит свои нейросети и подключится к тому же туннелю Cloudflare. Трафик доменов `api.youngfreak.click` и `media.youngfreak.click` автоматически перенаправится на Account B.
   - В завершение запуска Account B выложит файл-маркер `handover.signal` на Яндекс Диск.
6. **Выключение Account A**:
   - Таймер на Account A, опрашивая Яндекс Диск, обнаружит файл `handover.signal`.
   - Запустит финальный бэкап базы данных `agent.db` в облако.
   - Удалит маркер `handover.signal` с Яндекс Диска, запишет `/tmp/handover_complete` и погасит свои процессы.
   - Сессия Kaggle на Account A завершится, освобождая лимиты GPU.
7. **Результат**: Кластер продолжает бесшовно работать на Account B, сохранив историю вашей переписки в Telegram. Через 10 минут Account B аналогично запустит Account A.
