# Инструкция по настройке кластера 24/7 (Cloudflare Tunnels + Google Drive)

Данное руководство описывает настройку полностью защищенного ИИ-кластера с авто-ротацией на двух аккаунтах Kaggle с использованием постоянных туннелей **Cloudflare** и бесплатной облачной синхронизации через **Google Drive**.

Google Drive предоставляет 15 ГБ бесплатного пространства и не накладывает ограничений на подключение через Rclone (в отличие от Яндекс Диска, который требует платную подписку Яндекс 360 для работы по WebDAV).

---

## 1. Настройка внешних интеграций

### А. Telegram-бот (Интерфейс управления)
1. Найдите бота **@BotFather** в Telegram.
2. Создайте нового бота с помощью команды `/newbot`, задайте имя и скопируйте выданный API токен (например, `7932915257:AAFboG...`). Этот токен будет общим для всех нод кластера.

### Б. Личный Cloudflare Tunnel (Безопасный постоянный адрес)
1. Зарегистрируйте бесплатный аккаунт на [cloudflare.com](https://www.cloudflare.com/) и добавьте туда свой домен (например, `youngfreak.click`), сменив NS-серверы у регистратора домена на сервера Cloudflare.
2. Перейдите в боковую панель Cloudflare -> раздел **Zero Trust** -> **Networks** -> **Tunnels**.
3. Нажмите кнопку **Create a Tunnel**, введите имя (например, `kaggle-cluster`) и сохраните его.
4. На странице установки туннеля скопируйте буквенно-цифровой токен туннеля (длинный ключ после параметра `--token`). 
   * *Этот токен будет одинаковым для обеих нод Kaggle и прописывается в секреты.*
5. Перейдите на вкладку **Public Hostname** (Публичное имя хоста) в настройках вашего туннеля и добавьте два правила перенаправления:
   
   * **Правило 1 (Интерфейс API и Web UI)**:
     - Subdomain: `api`
     - Domain: `youngfreak.click`
     - Type: `HTTP`
     - URL: `localhost:8080`
   
   * **Правило 2 (Медиа-сервер картинок и видео)**:
     - Subdomain: `media`
     - Domain: `youngfreak.click`
     - Type: `HTTP`
     - URL: `localhost:8081`

### В. Авторизация Google Drive (Получение токена для Rclone)

Поскольку Kaggle работает автоматически, Rclone требуется предварительно сгенерированный токен для авторизации в Google Drive.

1. Скачайте и установите Rclone на свой компьютер с [rclone.org](https://rclone.org/downloads/).
2. Откройте командную строку/терминал на компьютере и выполните:
   ```bash
   rclone config
   ```
3. Выберите **`n`** (New remote) и укажите имя **`gdrive`**.
4. В списке типов хранилищ выберите **`drive`** (Google Drive).
5. Поля `client_id` и `client_secret` оставьте пустыми (нажмите Enter).
6. В пункте `scope` выберите **`1`** (Drive: Full access to all files).
7. Поля `service_account_file` и `Edit advanced config` пропустите (нажмите Enter).
8. В пункте `Use auto config?` выберите **`y`** (Yes).
9. Откроется браузер. Выберите свой аккаунт Google, нажмите **«Разрешить»** и закройте браузер после сообщения об успехе.
10. В терминале нажмите **`y`** для подтверждения сохранения.
11. Теперь откройте файл конфигурации Rclone на компьютере:
    * **Windows**: Нажмите `Win + R`, введите `%APPDATA%/rclone/` и откройте файл `rclone.conf` в блокноте.
    * **Linux / macOS**: Откройте файл `~/.config/rclone/rclone.conf`.
12. Найдите секцию `[gdrive]` и скопируйте **всю строку** после `token = `. Это JSON-строка вида:
    `{"access_token":"ya29...","token_type":"Bearer","refresh_token":"1//0...","expiry":"..."}`

## 2. Настройка секретов в Kaggle (Circular Chain)

Для создания циклической цепочки 24/7 потребуются два аккаунта Kaggle. Назовем их **Account A** и **Account B**.
1. Создайте на обоих аккаунтах новый Kaggle Notebook с именем `keglaai` (чтобы адрес ноутбука оканчивался на `/keglaai`).
2. В настройках профиля Kaggle каждого аккаунта в разделе **API** нажмите **Create New Token** для скачивания ключей `kaggle.json`.
   - Владельцу **Account A** понадобятся `username` и `key` соседа (**Account B**).
   - Владельцу **Account B** понадобятся `username` и `key` соседа (**Account A**).
3. В редакторе ноутбуков на обоих аккаунтах откройте меню **Add-ons > Secrets** и введите следующие секреты:

### Карта секретов Kaggle

| Имя секрета (Secret Key) | Значение на **Account A** | Значение на **Account B** | Описание |
| :--- | :--- | :--- | :--- |
| `TELEGRAM_BOT_TOKEN` | *Токен вашего Telegram-бота* | *Токен вашего Telegram-бота* | Общий токен управления |
| `HANDOVER_SECRET` | `youngfreak_secret` | `youngfreak_secret` | Секретное слово безопасности |
| `CLOUDFLARE_TUNNEL_TOKEN` | *Ваш токен туннеля Cloudflare* | *Ваш токен туннеля Cloudflare* | Токен, общий для обоих аккаунтов |
| `RCLONE_PROVIDER` | `gdrive` | `gdrive` | Провайдер бэкапа |
| `GDRIVE_TOKEN` | *JSON-строка из rclone.conf* | *JSON-строка из rclone.conf* | Авторизационный токен Google Drive |
| `HF_TOKEN` | *Ваш HuggingFace токен* | *Ваш HuggingFace токен* | Токен скачивания моделей |
| `ROTATION_TIME_SECONDS` | `600` | `600` | **Для теста**: 10 минут (600 секунд) |
| `NEXT_KAGGLE_USERNAME` | `username_от_Account_B` | `username_от_Account_A` | **Имя целевого аккаунта** |
| `NEXT_KAGGLE_KEY` | `api_key_от_Account_B` | `api_key_от_Account_A` | **API Ключ целевого аккаунта** |
| `NEXT_KAGGLE_SLUG` | `keglaai` | `keglaai` | Имя целевого ноутбука (slug) |

## 3. Сценарий проверочного 10-минутного тестирования

Чтобы убедиться в работоспособности синхронизации и авто-ротации:

1. **Запуск первой ноды**: Откройте ноутбук на **Account A** и нажмите **Run All** (Запустить всё).
2. **Инициализация**:
   - Ноутбук скачает репозиторий с GitHub.
   - Загрузит базу данных `agent.db` из папки `kaggle_sync` в вашем Google Drive (создаст чистую при первом запуске).
   - Подключит туннель Cloudflare Zero Trust. Домены `api.youngfreak.click` и `media.youngfreak.click` привяжутся к этой сессии.
3. **Проверка связи**:
   - Откройте Telegram и напишите боту команду `/status`. Вы должны получить ответ о зеленом статусе обоих портов.
4. **Ротация**: Фоновый таймер начнет отсчет 10 минут.
5. **Запуск Account B**:
   - Через 10 минут таймер на Account A выполнит `kaggle kernels push` на ноутбук соседа.
   - **Account B** запустится, скачает файл `agent.db` (с вашей перепиской и вайтлистом).
   - Подключится к тому же туннелю Cloudflare с тем же токеном. Cloudflare начнет балансировать трафик домена на новую ноду.
   - Account B выгрузит в папку `kaggle_sync` вашего Google Drive специальный файл-маркер `handover.signal`.
6. **Мягкое завершение Account A**:
   - **Account A** в фоновом режиме опрашивает ваш Google Drive. Обнаружив файл-маркер `handover.signal`, он:
     1. Выгрузит окончательную версию `agent.db` в облако.
     2. Удалит маркер `handover.signal` с Google Drive для следующего цикла.
     3. Мягко остановит свои процессы нейросетей, туннеля и создаст флаг `/tmp/handover_complete`.
     4. Ноутбук Account A автоматически прекратит выполнение ячейки, экономя ваши GPU-часы.
7. **Результат**: Трафик полностью переключился на **Account B**, история чата и права доступа сохранены, а баланс минут на первом аккаунте остановлен. Через 10 минут Account B запустит Account A в обратную сторону.
